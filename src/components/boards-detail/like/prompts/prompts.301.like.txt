핵심요구사항) 다음의 기능을 playwright 테스트를 활용하여 TDD기반으로 구현하고, 테스트에 통과할 때까지 반복할 것.
            1) 테스트 제외 라이브러리
                - jest
                - @testing-library/react

            2) 테스트 조건
                - timeout은 설정하지 않거나, 500ms 미만으로 설정할 것.
                - 모든 페이지의 로드 식별 조건은 아래를 따를 것.
                    - 페이지 로드 식별 요구사항: 고정식별자 data-testid 대기 방법
                    - 페이지 로드 식별 금지사항: networkidle 대기 방법

            3) 테스트 로컬스토리지 조건
                - 실제데이터를 사용할 것.
                - Mock데이터 사용하지 말 것.

            4) GraphQL 뮤테이션 타입
            mutation likeBoard($boardId: ID!) {
              likeBoard(boardId: $boardId)
            }

            mutation dislikeBoard($boardId: ID!) {
              dislikeBoard(boardId: $boardId)
            }


            5) 테스트시나리오(모든유저)
                1. /boards/detail/{boardId}에 접속하여 페이지 로드 확인
                2. 좋아요 버튼 존재 여부 확인
                3. 좋아요 버튼 클릭 → 좋아요 추가 확인
                4. 좋아요 개수 증가 확인
                5. 좋아요 버튼 재클릭 → 좋아요 취소 확인
                6. 좋아요 개수 감소 확인
                7. 하단 유저시나리오에 적절한 테스트시나리오를 생성하여 진행

핵심요구사항) 다음의 조건에 맞게 좋아요 기능을 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.
            1) 유저시나리오
            - 게시글 디테일 페이지에 좋아요 버튼이 표시될 것
            - 좋아요 버튼 클릭 시 좋아요 추가/취소 토글될 것
            - 좋아요 추가 시 하트 아이콘이 채워진 상태로 변경될 것
            - 좋아요 취소 시 하트 아이콘이 빈 상태로 변경될 것
            - 좋아요 개수가 실시간으로 증가/감소될 것
            - 낙관적 업데이트로 빠른 반응성 제공할 것
            - 서버 요청 실패 시 원래 상태로 롤백될 것

            2) 기술 요구사항
            - useMutation으로 likeBoard, dislikeBoard 구현
            - useState로 isLiked, likeCount 상태 관리
            - 낙관적 업데이트 패턴 적용
            - try/catch로 에러 처리 및 롤백 구현
            - 토글 방식으로 하나의 버튼에서 추가/취소 처리

            3) UI 요구사항
            - 좋아요 버튼에 하트 아이콘 표시
            - 좋아요 상태일 때: 채워진 하트 (heart-filled.svg)
            - 좋아요 아닐 때: 빈 하트 (heart.svg)
            - 좋아요 개수를 숫자로 표시
            - 호버 시 배경색 변경 효과
            - 좋아요 상태일 때 핑크색 강조 (#ff4081)
